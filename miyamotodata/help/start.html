Einleitung

Hallo, und Willkommen bei Miyamoto!

Du denkst wahrscheinlich: "Ich bin 12 Jahre alt und habe im Leben versagt, und ich kann nicht herausfinden, wie man einen so einfachen Editor wie Miyamoto benutzt! Wo kann ich hingehen, um den Leuten all meine dummen Fragen zu stellen, da ich nichts herausfinden kann?"

Fürchte dich nicht! Du brauchst niemanden zu fragen, all deine dummen, erbärmlichen, wertlosen Fragen können in diesem Dokument beantwortet werden. Lesen und navigieren Sie weiter durch die rot markierten Links, indem Sie mit dem Mauszeiger auf sie klicken. Wenn sich dies als zu schwierig erweist, versuche es mit der Anwendung Miyamoto! mit deiner Mutter.


Extrahieren der Dateien des Spiels

Um Miyamoto auszuführen, benötigen Sie den Ordner course_res des Spiels. Dies soll beweisen, dass du eine legale Kopie des Spiels hast.

Du kannst ddd verwenden, um die Dateien des Spiels zu extrahieren. Ein gutes Tutorial ist hier.

Sobald du die Spieldateien hast, öffne Miyamoto und wähle den Ordner course_res. Du kannst jetzt jedes Level im Spiel laden!


Mit Miyamoto starten

Wenn Sie eine vorkompilierte ausführbare Datei haben, klicken Sie einfach doppelt. Vorkompilierte ausführbare Dateien sollten als Windows exe, Mac-App oder Linux-Anwendung über cx_Freeze, Pyinstaller, Cython oder Nuitka verfügbar sein.

Für diejenigen, die eine der verfügbaren ausführbaren Dateien nicht ausführen können oder einfach nicht ausführen möchten, gibt es eine Quelldistribution. Die Quelldistribution benötigt die folgenden Pakete:

  • Python 3.4 or higher
  • PyQt 5.3 oder höher

  • Bitte beachten Sie: Linux- und Mac-Benutzer müssen WINE installiert haben!

Windows-Benutzer können mit dem Herunterladen des Windows-Binary-Installers für Python und des binären Installers für PyQt (erster und letzter Link) davonkommen, die Ihre Hand halten und Qt für dich installieren.

Mac OS X Snow Leopard-Benutzer müssen buildPyQt für beide Architekturen (i386 und x86_64) erstellen. PyQt erfordert auch eine Änderung der Datei configure.py - die i386-Zeile sollte ebenfalls x86_64 angeben. Wenn du eine ältere Version von Qt hast, musst du auf die neueste aktualisieren, und es wird dringend empfohlen, mit den Entwicklungsversionen von PyQt zu bauen.

Linux-Benutzer müssen PyQt5 über sudo pip3 install pyqt5 und WINE über sudo apt-get install wine für Ubuntu-Distributionen installieren lassen.

Wenn Sie ein anderes Betriebssystem oder eine Nicht-Ubuntu-Linux-Distribution haben, finden Sie es selbst heraus.

Zum Zeitpunkt des Schreibens wird Miyamoto für 32-Bit-Fenster nicht funktionieren, da puzzlehd und gtx_extract 64-Bit sind. Es gibt vielleicht einen älteren Build namens Miyamoto-SLOW, der funktionieren wird, aber wahrscheinlich in allem sehr fehlerhaft ist. Es ist auch super langsam, wie der Name schon sagt.


Neue Levels anlegen

Du musst noch eine beliebiges Level im Editor öffnen, bevor du das Menü benutzen kannst. Um eine neuen Level zu erstellen, wähle einfach "Neu" aus dem Menü Datei aus. Eine Ebene muss mindestens eine Zone und einen Eingang (ID 0) haben, sonst funktioniert das Level nicht. Sie sollten auch Tilesets hinzufügen, indem du auf "Slot X Tileset bearbeiten" klicken).


Öffnen vorhandener Dateien

Miyamoto! has two options for opening files.

"Level nach Name öffnen" erzeugt eine Liste aller Level in NSMBU und ermöglicht es dir, aus der Liste auszuwählen. Diese Option ist nützlich, um ein Gefühl dafür zu bekommen, wie man seine Level gestaltet, und um Referenzen nachzuschlagen.

"Level von Datei öffnen" ermöglicht es dir, eine Datei von deiner Festplatte auszuwählen und zu öffnen. Diese Option wird verwendet, um deine laufenden Arbeiten auszuwählen oder um die Kreationen anderer zu überprüfen. Es unterstützt sowohl .szs als auch.sarc Dateitypen.
